# Configuration

All runtime configuration lives in `{data-dir}/config.toml`. The
default data directory is `%APPDATA%/vrhub-server` on Windows and
`$HOME/.vrhub-server` on Unix. Override it with the `-data-dir` CLI
flag or by setting the data dir in the admin UI (**Configuration**
widget).

The full configuration surface is defined in
[`pkg/types/types.go`](../pkg/types/types.go) and consumed by
[`internal/config/config.go`](../internal/config/config.go). The
sections below mirror that file.

## File layout

```toml
# Top-level keys
data_dir      = "C:\\Users\\you\\AppData\\Roaming\\vrhub-server"
game_folders  = ["D:\\VR\\library", "D:\\VR\\sidequests"]

[server]
host = "127.0.0.1"
port = 39457
mode = "normal"          # "normal" or "setup"

[database]
path = "C:\\Users\\you\\AppData\\Roaming\\vrhub-server\\vrhub.db"

[metadata]
url               = "https://raw.githubusercontent.com/nicnacnic/MetaMetadata/main/dataset.json"
refresh_interval  = "24h"

[update]
enabled        = true
check-interval = "24h"
auto-apply     = false
github-token   = ""
owner          = "LeGeRyChEeSe"
repo           = "vrhub-server"

[admin]
username          = "admin"
password_hash     = "$2a$10$..."          # bcrypt; set by the wizard
archive_password  = "your-cleartext-archive-password"
api_key_hash      = "<sha256-hex-of-your-api-key>"
```

## Top-level keys

### `data_dir` (string)

Absolute path to the data directory. Normally written by the setup
wizard; rarely edited by hand. Defaults to the platform-specific path
listed above.

### `game_folders` (array of strings)

Absolute paths the scanner walks for APK and OBB files. Editable from
the admin UI; re-scanning is automatic.

## `[server]`

| Key    | Type   | Default       | Description                                   |
|--------|--------|---------------|-----------------------------------------------|
| `host` | string | `127.0.0.1`   | Listen address. Set to `0.0.0.0` for LAN.     |
| `port` | int    | `39457`       | TCP port to listen on.                        |
| `mode` | string | `normal`      | `normal` or `setup`. Forced to `setup` if the archive password is missing. |

## `[database]`

| Key    | Type   | Default     | Description                                  |
|--------|--------|-------------|----------------------------------------------|
| `path` | string | `vrhub.db`  | Absolute or data-dir-relative SQLite path.   |

## `[metadata]`

| Key                | Type          | Default                              | Description                                |
|--------------------|---------------|--------------------------------------|--------------------------------------------|
| `url`              | string        | MetaMetadata dataset URL             | JSON source for game metadata enrichment.  |
| `refresh_interval` | Go duration   | `24h`                                | Background refresh cadence.                |

A Go duration is a signed sequence of decimal numbers, each with
optional fraction and unit suffix: `"300ms"`, `"1h30m"`, `"24h"`.

## `[update]`

| Key              | Type          | Default          | Description                                              |
|------------------|---------------|------------------|----------------------------------------------------------|
| `enabled`        | bool          | `true`           | Toggle the GitHub releases checker.                      |
| `check-interval` | Go duration   | `24h`            | How often to poll.                                       |
| `auto-apply`     | bool          | `false`          | Apply updates automatically. **Disabled by default.**    |
| `github-token`   | string        | (empty)          | Optional PAT to lift the 60 req/h unauthenticated cap.    |
| `owner`          | string        | `LeGeRyChEeSe`   | GitHub owner used for release lookups.                   |
| `repo`           | string        | `vrhub-server`   | GitHub repository name.                                  |

## `[admin]`

| Key                | Type   | Description                                                                 |
|--------------------|--------|-----------------------------------------------------------------------------|
| `username`         | string | Admin web UI login.                                                        |
| `password_hash`    | string | bcrypt hash of the admin password. Regenerated by the wizard.              |
| `archive_password` | string | **Cleartext** password used to encrypt `meta.7z` with AES-256. Required because the VRHub client needs the original password to decrypt. |
| `api_key_hash`     | string | SHA-256 hex digest of the admin API key. The plaintext is held in process memory only. |

### Why is `archive_password` in cleartext?

`meta.7z` is encrypted with AES-256 so a casual scanner of the data
directory cannot extract the game list. The VRHub client must be able
to decrypt the archive on-device, which means the original password
has to leave the server &mdash; either embedded in `config.json` for
manual entry, or read from the operator-provided JSON. Hashing the
password on disk would make the archive permanently unreadable.

The mitigation is filesystem-level: restrict the ACL on the data
directory, run the server behind a reverse proxy, and never copy
`config.toml` to a less-trusted host.

## CLI overrides

Two CLI flags take precedence over `config.toml`:

| Flag        | Effect                                                     |
|-------------|------------------------------------------------------------|
| `-data-dir` | Override the data directory entirely.                      |
| `-port`     | Override `[server].port` for this run only.                 |

`-host` is intentionally not exposed; the security guidance is to
keep the default loopback bind and put a reverse proxy in front when
LAN reachability is needed.

## First-run detection

If `{data-dir}/config.toml` does not exist, the server starts in
**setup mode**:

- All public routes return `503 Service Unavailable` with a
  `Retry-After: 0` header.
- All `/admin/*` routes (except `/admin/setup`) redirect to
  `/admin/setup`.
- The wizard is reachable at `/admin/setup`.

The wizard writes `config.toml` and restarts the server in normal
mode on completion. A pre-existing file that fails to parse triggers
a fatal log and the server exits &mdash; this is intentional, we
never silently fall back to defaults.

## Reloading

Configuration changes made through the admin UI (game folders,
metadata URL, update interval, etc.) are persisted to
`config.toml` and reloaded in-process. Some keys (`[server]`,
`[admin].username`, `[admin].password_hash`) require a server
restart to take effect.
